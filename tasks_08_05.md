

## Задачи на инкапсуляцию 

### Задача 1. Банковский счёт

Создайте класс `BankAccount`, который полностью скрывает внутреннее состояние.

**Требования:**
- Приватное поле `__balance` (баланс).
- Приватное поле `__owner` (владелец счёта).
- Приватное поле `__pin` (пин-код для операций).
- Публичные методы:
  - `deposit(amount, pin)` – пополнение, но только если пин верный.
  - `withdraw(amount, pin)` – снятие, если пин верный и достаточно средств.
  - `get_balance(pin)` – возвращает баланс только при правильном пине.
- Приватный метод `__validate_pin(pin)` – проверяет корректность пина.
- Приватный метод `__is_positive(amount)` – проверяет, что сумма > 0.

**Цель:** убедиться, что ни баланс, ни владелец, ни пин нельзя прочитать или изменить напрямую.

### Задача 2. Термометр с историей (инкапсуляция + property)

Создайте класс `Thermometer`.

**Требования:**
- Приватное поле `__celsius`.
- Приватное поле `__history` – список всех значений температуры, которые когда-либо устанавливались.
- Свойство `celsius` (геттер и сеттер):
  - При установке новое значение не может быть ниже -273.15.
  - Каждое новое значение автоматически добавляется в `__history`.
- Свойство `fahrenheit` (только геттер) – возвращает температуру в фаренгейтах.
- Публичный метод `get_history()` – возвращает копию истории (чтобы внешний код не мог её изменить).

**Цель:** скрыть историю и прямой доступ к `__celsius`, но предоставить контролируемый интерфейс через property.

## Задачи на наследование 

### Задача 3  

Создайте базовый класс `Animal` с методом `make_sound()`, который выводит "Some animal sound".

Создайте классы-наследники:
- `Dog` – переопределяет `make_sound()` на "Woof! Woof!".
- `Cat` – переопределяет `make_sound()` на "Meow!".
- `Cow` – переопределяет `make_sound()` на "Moo!".

Добавьте в каждый класс метод `move()`, который возвращает способ передвижения (у собак – "runs", у кошек – "walks silently", у коров – "walks slowly"). В базовом классе `move()` может быть абстрактным или возвращать "moves".

**Требование:** создать список из разных животных и в цикле вызвать у каждого `make_sound()` и `move()`.

### Задача 4  
Создайте базовый класс `Vehicle`:
- Конструктор принимает `brand` и `speed`.
- Метод `info()` возвращает строку `"Brand: {brand}, Speed: {speed} km/h"`.
- Метод `honk()` возвращает `"Beep beep!"`.

Создайте класс `Car` (наследник `Vehicle`):
- Добавляет поле `doors` (количество дверей).
- Переопределяет `info()` – добавляет информацию о дверях.
- Добавляет метод `open_trunk()`.

Создайте класс `Motorcycle` (наследник `Vehicle`):
- Добавляет поле `has_sidecar` (есть ли коляска).
- Переопределяет `honk()` – возвращает `"Beep! (louder)"`.
- Добавляет метод `wheelie()`.

---

Вот теперь порядок правильный: сначала две задачи про сокрытие данных (инкапсуляция), потом две про расширение классов (наследование).
